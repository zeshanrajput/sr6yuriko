## What Does the Code Do?

The Kechibi Code may be the first wave in a new stage of programming development, something that draws upon the Matrix in all its intricacies and directs them toward a very specific goal— in this instance, manipulating financial markets.

**Load the Program** . This is a complicated program and will strive to consume any device’s program array. It needs this setup in order to reach maximum function. Cyberdecks, cyberterminals, dedicated hosts, and certain Monads and technomancers can operate the program.

**State a goal and acceptable parameters** . This can’t just be “make money.” The code operates by making a series of parallel transactions, not only buying on margin or selling short on stocks that are susceptible to panics or other rapid price changes, but also planting a number of social manipulations to reinforce what it does through purchases. Fake news stories, social media posts, and more are part of its functioning. This works best when it complements real-world information instead of trying to work against it. Once it starts manipulating prices, savvy investors should buy or sell targeted equities so that they benefit from the manipulations. If the code does not have a target and basic instructions about what damage to inflict, it is ineffective.

**Activate the code** . This is where the users of the code mostly sit back and watch what happens. The processing and actions that take place do so at a rapid speed that is simply not suitable for game play.



The whole point of the code is that it can act faster than any individual or group can keep up with.

**Give it time** . Seriously, just let it do its thing. Start to watch your location. The code will reach a point where it will start to send warnings if it detects actions taken to stop its operations.

**Keep it safe** . So, here’s the thing: No corporate financial officer is going to let stuff like this go unnoticed. The longer it operates, the more likely it is that someone will notice something is up. Matrix security supervising trading should make an Electronics + Intuition test to see if they notice something odd happening. The threshold for this test and the frequency it must be rolled are variable, depending on where activities take place and how long they have been going on. When the code is running on non-rated and private targets, the threshold is 4, and the test is rolled once every 10 minutes. For A-rated corporations, the threshold is reduced to 3, and the interval is reduced to once every 2 minutes. For AA corporations, the threshold is 2 and the interval is once every minute, while AAA megacorporations have a threshold of 1 and a chance to notice this every 5 combat rounds (about 15 seconds). The code doesn’t much care if it’s noticed. It keeps trying to run transactions until it is hit by any attack that would cause it damage; once that happens, it self-destructs.

The code will draw attention from high-ranking GOD agents, so if the PCs are anywhere near an operating code, they should be willing to deal with the best GOD can send. If they don’t want to deal with that, they should probably enforce some separation.

In general, the code should not benefit the player characters. Its programming is at a very high


level, and it tends to route money through a series of accounts so that no one knows where it will end up—and it’s really hard to get it to end up somewhere the designers don’t want it to be.
