## **Trap Doors**

**Over the years, the term trap door has described an assortment of different constructs. Today, it is generally used to describe a nested host connected to a killing jar. These small hosts do not have any active IC and the only concealed gates present lead in and out of the killing jar. The host attached to a trap door is always generated by a framework host with an automated kill switch. Once an intruding icon enters the trap door, the kill switch cuts the power to the framework host, dumpshocking any intruder inside the trap door.**
