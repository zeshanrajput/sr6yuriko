## **Membership in a Data Haven**

**A data haven uses the regular system of Favor Points for a group contact (p. 176,** _**Sixth World Companion**_ **), with the group Loyalty setting the number of Favor Points available for the month. In addition to the normal pool of Favor Points, posting free paydata on a data haven generates 1 extra Favor Point for every 3,000 nuyen in value. Withdrawing paydata for free uses Favor Points at the same rate. Favor Points gained or used from paydata do not reset at the end of the month, as**



**opposed to the regular pool of Favor Points. These Paydata Favor Points are carried over from month to month.**

**In addition to the Favor Points, all data haven members have a reputation score based on the Loyalty rating and any additional Favor Points from paydata. The base reputation is the current Loyalty rating times a fxed multiplier set by the gamemaster (for example, JackPoint is 50 and the Nexus is 30) and represents active involvement with the group. An additional number of points of rep, using the same multiplier, is added for each Favor Point worth of accumulated paydata held. Services for, or actions against, the data haven operators and members can also increase or decrease the rep score by an additional (positive or negative) number of points without changing the current number of Favor Points.**

**That makes a data haven reputation score = (Loyalty x Multiplier) + (unspent Paydata Favor Points x Multiplier) + action score adjustment. For the Denver Data Haven specifcally, the rep score would be (Loyalty x 30) + (unspent Paydata Favor Points x 30) + action score adjustment.**
