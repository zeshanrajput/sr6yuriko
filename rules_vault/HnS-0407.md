## Emulation

While an AI can be deviceless, when they want to run cyberprograms, they do so using an Emulate action. AIs use emulation to borrow some of the raw computing power provided by the device network that generates the grids; they use this power to mimic the specific software they need to perform a Matrix action. GOD and the corps aren’t completely against emulation, since it doesn’t impact the performance of the Matrix or any connected devices, but they’re quick to punish Matrix entities that use emulation for criminal activity. AIs are allowed to use emulation for legal cyberprograms on the public Matrix without accumulating an overwatch score. Devices and hosts extend this courtesy to AIs with legal access. An AI character can emulate a number of cyberprograms equal to their Data Processing.

If the AI is emulating illegal or hacking cyberprograms, then their overwatch score increases by 1 point per turn for each illegal cyberprogram. If any illegal actions are performed using emulated cyberprograms, the AI character’s overwatch score increases for each cyberprogram they are currently emulating.

When an AI reaches convergence, grids and hosts actively try to stop them from using emulation instead of increasing the character’s overwatch score. To do this, they throw malicious code into the processing power AIs are using for emulation. When an AI uses the Emulate action during convergence, the character must resist an amount of Matrix damage equal to the number of the emulated cyberprograms active. This damage is resisted with Willpower + Firewall.
