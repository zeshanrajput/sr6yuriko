## Host Scale

Host scale is documented in Hack and Slash on p. 51. The geographic footprint of the host is relative to its size. The scale 6 hosts of the megas hang far above the horizon of the open Matrix, visible from anywhere in the world, while the scale 1 host at the corner grocery is only visible from the street in front of it. While it’s a fact that both hosts can be accessed from anywhere in the world through the Matrix, the smaller scale 1 host will be harder to find if you don’t know exactly where to look. As a practical matter, scale also represents the number of active users a host can handle simultaneously—there’s just no reason to design the local Stuffer Shack host to handle 10,000 users at one time. For the purposes of Killing Jars and Trap Doors (Falling Point, p. 66-67), these hosts have a Scale of 1 for two reasons. First, they’re meant to be a temporary space for a limited number of active users before they either log in, or they’re dumped out. Second, they’re also meant to be hard to find on the Matrix—in fact, most of these are so well hidden you have to know exactly where they are to access them.

Queries use the Matrix Search rules but gain one additional hit, along with the normal hits rolled during the test. So, if the test results in 2 net hits, use the result for 3 hits on the Legwork table. If there are no hits, no information is obtained. This test can still result in a glitch or critical glitch. The query system is built to use some combination of name, location, service type, or image.
