## **New Host Architecture Gates**

**Gates are simply the network connection points between nested hosts. Gates that are designed for public or general employee use are obvious and can be found without a Matrix Perception test. They require nothing more than proper access to the connecting host and an Enter Host minor action.**

**Binary gates, also known as air gaps, are designed to be turned on and off. This can be on a regular schedule, by remote command, or from inside the inner nested host. They are only usable in the powered state—when unpowered, they cut access to a nested host with no connection existing between the nested hosts. Unless there are other connections to the outside Matrix or to other nested hosts with Matrix access, an unpowered binary gate creates a de facto offine host, dumping out anyone without local access from inside the air gap. A normally closed binary gate is the primary method used by most offine hosts to connect to the Matrix— through another host—for intermittent data transmission. Note that while Foundation hosts can be separated from the upper Matrix, access can still be gained through an anchor in the host Foundation or through the host Foundation’s null node.**

**Concealed gates are another security feature available for nested hosts. If you have an idea that such a gate exists and roughly where it should be, the concealed gate can be perceived using a Matrix Perception test with the threshold set by the gamemaster according to the programming skill of whoever designed it. Once a concealed gate is**

**perceived, it can be used in the same way as any other gate. If multiple concealed gates exist in a host, each one requires a separate Matrix Perception test to perceive. Concealed gates may be programmed with a password that, when transmitted, makes the gate obvious to the password holder and can then be used as a basic gate.**

**Without a password, or knowing about where to look, the host structure may be analyzed to fnd the likely locations of any concealed gates. These can be detected in the same way as fnding a Foundation Portal and require an Electronics + Logic (host rating, 1 combat turn) Extended test to discover the most likely locations. After these are deduced, a Matrix Perception test is still required to perceive each concealed gate.**
