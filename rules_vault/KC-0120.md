## decaying kechibi code

The designers of the Kechibi Code don’t want it just sitting around to be used whenever certain people feel like it, and they certainly don’t want multiple copies proliferating all over the place. The code has a built-in auto-decay function, and this decay accelerates as it is used, copied, or taken offline.

The decay is mild at first, and the code simply becomes a little slower and less efficient in executing its price manipulations, purchases, and sales. Then it becomes less optimized—it still generates profits, but they are lesser than before, eventually getting to the point where they don’t beat market indexes. Shortly after that, the code breaks, and it is no longer capable of executing trades.

Deckers, technomancers, and AIs may attempt to analyze a decayed copy of the code for hints about what it does. They should roll an Electronics + Logic (2) test; the gamemaster can raise a threshold if they think a copy of the code experienced extra decay. Admin access to whatever device the code is stored on is needed to make this test. The table below lists what they learn based on the number of net hits.
