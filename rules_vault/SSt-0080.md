## **Chips for Brains**

**But let’s get through the bad stuff frst—the downsides. There are two big ones that I’m constantly aware of. The frst downside is that drones are pretty fragile—even the heavy combat drones aren’t able to withstand sustained fre. The other downside is that dog-brains—the internal pilot - systems—are pretty dumb. No matter how diff cult the task you give them, they’ll do their best to get it done, but on their own their best is not great. They tend to take instructions very literally, and don’t have much in the way of creativity or improvisation. This can be overcome, but it takes expensive upgrades and software and a strong familiarity with how they operate.**

**Most stock dog-brains are pretty basic. Even the better stock models can’t match metahuman intelligence. They can be upgraded to expert systems that approach true AI, but this is very expensive—and since drones are so fragile, it’s usually not worth the investment. The best way to compensate for this is using autosofts. A drone can only run one or two autosofts on its own, but I would only recommend this as a backup or for certain very specifc purposes. The best way to improve a dog-brain’s functionality is to slave it to an RCC so it can run autosofts from that. This overcomes the limitation of only being able to run one or two autosofts per drone, though most RCCs can only run two or three times that number. That’s why I almost always run Virtual Machine—sure, it increases Matrix damage a little, but the extra program slots are defnitely worth it, especially since you’re always going to have other uses for those program slots.**
